## Script Demo 

1. The tool we chose to use for the project is Retool, which requires us to pay for any kind of authentication. Therefore, we don't have authentication.
2. So, our homepage is a list of existing projects and an option to create a new project.
   1. * Click on the 'New Project' button
   2. * Project title (e.g., 'Demo Presentation')
   3. * Fill start and end dates
   4. * Fill description (e.g., 'This is a demo presentation')
   5. * Click on the 'Submit' button
3. We return to the homepage where we can now see the project we just created.
4. If we wanted to, we could create another project.
   1. * Click on the 'New Project' button
   2. * Click on the 'Cancel' button
5. If we click on the project we just created, we go to the project page where we can see the product backlog, sprints, team members, and labels.
6. We start by creating a new user.
   1. * Scroll down
   2. * Fill in the name (e.g., 'Ana Almeida')
   3. * Click on the '+' button
   4. Create 1 or 2 more users to have options when creating tasks.
   5. Assign one of the users as the scrum master.
      1. * Click on the tag that says 'Team Member'
7. We create some labels.
   1. * Fill in the name (e.g., 'Bug')
   2. * Choose the color (e.g., 'red')
   3. * Click on the 'add label' button
   4. Create 1 or 2 more labels to have options when creating tasks (e.g., 'Feature' and 'Improvement').
8. We create a new task.
   1. * Click on the 'New Task' button
   2. * Fill in the title (e.g., 'Create a new task')
   3. * Fill in effort (e.g., '1')
   4. * Fill in priority (e.g., '3')
   5.  * Fill in description (e.g., 'This is a demo task')
   6.  * Click on the 'Submit' button
   7.  Create 1 or 2 more tasks with different priorities (e.g., 5 and 1) to show that tasks are ordered by priority.
9. We create a new sprint.
    1. * Click on the 'New Sprint' button
    2. * Fill in the title (e.g., 'Demo Sprint')
    3. * Fill in start and end dates
    4. * Fill in description (e.g., 'This is a demo sprint')
    5. * Click on the 'Submit' button
10. Edit task to assign it to the sprint.
    1. * Click on the Edit button
    2. * Choose the sprint
    3. * Click on the 'Submit' button
    4. Return to the previous page (project page) and do the same for another task.
11. Edit task to assign users.
    1. * Click on the Edit button
    2. * Choose the users
    3. * Depending on our decision regarding the task creation order, adding users and labels might also be necessary.
    4. * Click on the 'Submit' button
12. Move task to 'ongoing'.
13. Move task to 'completed'.
14. Ask to observe the chart.
15. Repeat the same for the other task.
