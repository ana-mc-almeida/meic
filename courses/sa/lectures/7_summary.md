## Tactics for Modifiability

- **Cohesion**: Is the degree to which the elements inside a module belong together.
- **Coupling**: Is the degree to which the elements inside a module depend on each other.

### Increase cohesion

- **Split modules**: Split modules into smaller ones.
- **Redistribute responsibilities**: Redistribute responsibilities among modules.

### Reduce coupling

- **Encapsulate**: Encapsulate the module's internal state.
- **Use an intermediary**: Use an intermediary to communicate between modules.
- **Abstract common services**: Abstract common services to a separate module.
- **Restrict dependencies**: Restrict dependencies between modules.
